# 🎨 [디자인 가이드] 다정산 (Dajeongsan)

## 1. 디자인 컨셉 & 컬러

- **톤앤매너**: 따뜻하고 다정한 미니멀 디자인. 정산 요청이 "돈 독촉"이 아니라 "배려가 담긴 요청"으로 느껴지도록, 기존 정산 서비스의 차갑고 계산적인 핀테크 인상을 피한다. 금액 정보는 명확하고 신뢰감 있게, 배려 요소(절삭 혜택·배려 문구·배려 기록)는 부드럽고 감성적으로 표현한다.
- **테마 모드**: 라이트 모드 고정. 지불자가 카톡 링크로 처음 진입하는 화면이므로 밝고 환영하는 인상이 필수. 배경은 순백 대신 따뜻한 웜 뉴트럴(`#FAFAF9`, `bg-stone-50`)을 기본으로, 카드는 흰색(`#FFFFFF`, `bg-white`).
- **포인트 컬러 (Primary)**: 따뜻한 코럴 로즈 계열 (`#F43F5E`, `bg-rose-500`, 호버 `bg-rose-600`). 주요 CTA([정산 요청하기], [계좌번호 복사하기] 등)와 절삭 토글 ON 상태, 배려 모드 관련 강조에 사용. "다정함"을 전달하는 서비스 정체성 컬러.
- **보조 컬러 (Secondary/Accent)**:
    - 배려/혜택 하이라이트: 로즈 틴트 (`#FFF1F2`, `bg-rose-50` / 텍스트 `text-rose-600`) — 절삭 토글 카드, "지금까지 받은 배려" 배너, 배려 문구 영역의 배경.
    - 완료/확인 상태: 에메랄드 계열 (`#10B981`, `text-emerald-600`, 뱃지 `bg-emerald-100 text-emerald-700`) — [확인함] 상태 뱃지, 복사 성공 토스트 아이콘.
    - 미확인/보조 정보: 스톤 계열 (`#78716C`, `text-stone-500`, 뱃지 `bg-stone-100 text-stone-500`).
- **금지 사항**: 차가운 블루/인디고 계열을 주조색으로 사용 금지(기존 정산 서비스와의 차별화 포인트). 채도 높은 원색 배경 금지. 금액·독촉성 정보에 경고성 레드(`red-600` 이상) 사용 금지 — 에러 표시도 부드러운 톤(`text-rose-500` + 다정한 안내 문구)으로 처리. rose/emerald/stone 3계열 외 컬러 혼용 금지.

## 2. 레이아웃 & 타이포그래피

- **기준 화면**: 모바일 퍼스트 (`max-w-md mx-auto` 중앙 정렬, 좌우 `px-4~5`). 모든 화면(생성·공유·참여자 확인·대시보드)이 카카오톡 인앱 브라우저·모바일 세로 화면 기준으로 깨지지 않아야 함. 데스크톱에서는 동일 레이아웃을 중앙 정렬로 유지.
- **카드/컨테이너 스타일**: 부드러운 라운딩(`rounded-2xl`)과 은은한 입체감(`shadow-sm ring-1 ring-stone-100`). 참여자별 카드 리스트(3-2 공유, 3-4 대시보드)는 이 스타일을 공통 사용. 절삭 토글 카드와 "지금까지 받은 배려" 배너는 `bg-rose-50 rounded-2xl`로 일반 카드와 시각적으로 구분해 배려 요소임을 드러낼 것.
- **폰트**: Pretendard, 시스템 폰트 폴백 포함 (`font-family: Pretendard, -apple-system, "Apple SD Gothic Neo", sans-serif`). 금액 숫자는 `tabular-nums`로 자리 흔들림 방지.
- **텍스트 위계**:
    - 화면 타이틀("오늘 모임, 얼마씩 정산할까요?" 등): `text-2xl font-extrabold text-stone-900`
    - 서브 문구("금액은 정확하게, 요청은 조금 더 다정하게 도와드릴게요" 등): `text-sm text-stone-500`
    - **확정 금액은 화면 내 최상위 위계**: `text-4xl font-extrabold text-stone-900` (예: "26,000원만 보내면 돼요")
    - 절삭 전 원래 금액: 취소선 + 회색 (`line-through text-stone-400 text-base`)으로 배려받은 차이가 한눈에 보이게
    - 절삭 금액·배려 문구("437원이 덜어졌어요", "끝자리는 요청자가 덜어줬어요!"): `text-rose-600 font-semibold`
    - 라벨·보조 문구: `text-xs text-stone-400`

## 3. 컴포넌트 & 인터랙션 (Tailwind CSS)

- **버튼**:
    - 주요 CTA: 풀너비 `w-full h-12 rounded-xl bg-rose-500 text-white font-semibold`, 호버 `hover:bg-rose-600 transition-all`, 클릭 시 `active:scale-95`.
    - 비활성화 상태(총액 미입력·참여자 0명): `bg-stone-200 text-stone-400 cursor-not-allowed` + 버튼 위에 명세의 안내 문구("정산할 금액을 입력하면 요청을 만들 수 있어요" 등)를 `text-xs text-stone-400`으로 노출.
    - 보조 버튼([링크 복사], [다시 공유], [건너뛰기]): 아웃라인 스타일 `border border-stone-200 bg-white text-stone-700 rounded-xl`.
    - [카카오톡으로 보내기]: 카카오 브랜드 옐로우(`#FEE500`, `bg-[#FEE500] text-stone-900`)만 예외적으로 허용.
- **절삭 토글(빠른 정산 배려 모드)**: 기본 ON. 스위치 OFF `bg-stone-300` → ON `bg-rose-500` 전환. ON 시 미리보기 문구("예) 26,437원 → 26,000원만 보내주세요 :)")가 fade-in으로 등장.
- **은행 선택란**: 계좌번호 입력란과 분리된 필드. 미선택 시 "은행 선택" placeholder, 선택 후 `로고 + 은행명` 표시. 클릭 시 바텀시트(`rounded-t-2xl bg-white shadow-xl`)에 은행 목록을 그리드로 노출 — 각 항목은 로고 + 은행명, 최소 44px 터치 영역. 바텀시트 상단에 검색바(`h-12 rounded-xl border-stone-200`, 은행명 실시간 필터링)를 두어 목록 스크롤 없이 검색으로도 선택할 수 있게 한다. 로고 이미지는 외부 URL 대신 인라인 SVG/이니셜 원형 아이콘(은행 브랜드 컬러 배경)으로 구현해 깨진 리소스를 방지.
- **상태 뱃지(대시보드)**: [확인함] `bg-emerald-100 text-emerald-700 rounded-full px-2.5 py-0.5 text-xs font-medium` / [미확인] `bg-stone-100 text-stone-500` 동일 형태.
- **토스트**: 하단 고정(`fixed bottom-6 inset-x-4`), `bg-stone-900 text-white rounded-xl px-4 py-3 text-sm`, fade-in-up 등장 후 2초 뒤 자동 소멸. 계좌 복사·링크 복사·공유 실패 폴백 안내에 공통 사용.
- **인앱 계산기**: 총액 입력 필드 **왼쪽**에 계산기 버튼(🧮 아이콘 + "계산기 사용" 라벨)을 **상시 노출**해 필드 클릭 전에도 계산 기능의 존재를 인지시킨다. 버튼 스타일은 안정적인 아웃라인 톤(`border border-stone-200 bg-white text-stone-600 rounded-xl`, 호버 시 `border-rose-300 text-rose-500`)으로 입력 필드와 나란히 있어도 차분하게 어울리도록 한다. 버튼 클릭 시 바텀시트 형태(`rounded-t-2xl bg-white shadow-xl`)로 열리며, 키패드 버튼은 최소 44px 확보. 계산 결과는 총액 필드에 자동 입력.
- **핵심 애니메이션**:
    - 절삭 적용 결과 화면(3-3): 원래 금액이 먼저 보인 뒤 취소선이 그어지고 확정 금액이 카운트 애니메이션으로 내려가는 연출 — 배려받는 순간을 서비스의 하이라이트로 만들 것.
    - 화면 진입 시 카드들이 순차 fade-in-up (`opacity-0 translate-y-2` → 표시, 각 80ms 딜레이).
    - 화면 전환은 부드러운 페이드.
- **피드백 상태**: 로딩(링크 생성·데이터 조회 중)은 스피너 + "정산 요청을 준비하고 있어요" 류의 다정한 문구, 성공은 토스트, 실패 시에도 화면이 깨지지 않고 재시도 안내를 부드러운 톤으로 노출(외부 API·DB 폴백 원칙 준수). 빈 상태 문구(참여자 0명, 배려 기록 없음)는 명세의 문구를 그대로 사용하고 이모지 1개를 곁들여 허전하지 않게 처리.

## 4. 품질 기준 (오류 방지 규칙 — 그대로 유지 권장)

> 💡 이 섹션은 어떤 프로젝트든 공통 적용되는 안전장치입니다. 삭제하지 않는 것을 권장합니다.

- Tailwind 클래스는 **완성된 문자열 리터럴**로 작성할 것. (동적 조합 문자열은 스타일 누락의 원인)
- 색상·간격은 임의 값 남발 대신 Tailwind 기본 스케일을 우선 사용할 것.
- 터치 대상(버튼 등)은 충분한 크기(세로 44px 이상)를 확보할 것.
- 명도 대비를 지켜 어두운 배경 위 어두운 텍스트 같은 가독성 문제가 없도록 할 것.
- 이미지·아이콘이 준비되지 않은 경우 외부 URL 대신 이모지 또는 인라인 SVG로 대체할 것. (깨진 리소스 방지)
- 애니메이션은 CSS 기반으로 구현하고, 콘텐츠 확인을 방해할 만큼 과하게 넣지 말 것.
