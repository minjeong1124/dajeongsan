## 3. 화면별 상세 명세 (UI/UX)

### 3-0. 메인 랜딩 (`/`)

**화면 목적:** 방문자가 "내 얘기다"라고 느끼게 만들고, 의심을 하나씩 걷어내 정산 요청 생성(`/create`)으로 전환시키는 설득 흐름 페이지. 제품 설명서가 아니라 설득 흐름으로 구성한다.

**설득 흐름 (사용자가 순서대로 느껴야 하는 감정):**

1. 공감 — "맞아, 정확히 정산 요청하는 거 은근 민망했어." → Hero(결과 중심 헤드라인 "쪼잔해 보일 걱정 없이 정산 요청을 보내세요") + 공감 순간 3가지 카드
2. 해결 시연 — "26,437원이 26,000원이 되는 식이면 느낌이 훨씬 부드럽네." → Before/After 요청 문구 비교 카드
3. 독촉이 아닌 이유 — "빠르게 확인하면 혜택이 생기니까 독촉처럼 안 보이겠다." → 시간대별 혜택 표(1시간/5시간/이후)
4. 상호성 — "배려를 주면 상대도 배려해주겠지, 나만 손해는 아니야." → 배려 기록 예시 카드 + 근거 수치(회당 최대 999원, 99원씩 3개월 11회 정산 시 약 1,089원, 5,000원 미만 미적용)
5. 행동 — "한 번 만들어볼까?" → 시작 3단계(장벽 제거: 회원가입·앱 설치 불필요) + Final CTA

**Final CTA:**

- 섹션 제목 : 오늘 정산도 조금 더 다정하게 보내볼까요?
- Sub Copy : 정확한 계산은 그대로,\n요청의 부담은 조금 덜어보세요.
- CTA : [정산 요청 만들기] → `/create` 이동

**Footer:** 너무 많은 정보를 넣지 않고 서비스명과 짧은 카피만 남긴다.

- 다정산
- 정산도 조금 더 다정하게

**라우트 참고:** 랜딩 추가에 따라 정산 요청 생성 화면은 `/` → `/create`로 이동. 대시보드·공유 화면의 [새 정산 만들기] 류 CTA는 `/create`로, 참여자 화면 하단의 [나도 다정하게 정산해보기]는 서비스 소개가 필요한 신규 사용자이므로 랜딩(`/`)으로 연결.

### 3-1. 정산 요청 생성 (`/create`)

**화면 목적:** 정산 요청자가 모임의 총 금액, 참여자, 정산 방식을 입력하고 빠른 정산 배려 모드를 선택해 참여자별 정산 요청 링크를 생성하는 화면

**핵심 문구:** 타이틀 "오늘 모임, 얼마씩 정산할까요?" / 서브 "금액은 정확하게, 요청은 조금 더 다정하게 도와드릴게요"

**주요 요소:**

- 내 정보 (요청자 정보) 입력
    - 저장/재노출 규칙 : 정산 요청 생성 시 내 정보 영역의 값(이름·은행·계좌번호)이 저장되고, 이후 해당 사용자가 재접속하면 디폴트 값으로 자동 입력되어 노출됨
    - 이름 입력 (placeholder : "내 이름을 입력해주세요")
    - 은행 선택란 + 계좌번호 입력란 — 분리된 2개 필드
        - 은행 선택란 : 클릭 시 한국 주요 은행 목록이 **로고 + 은행명**으로 노출되는 선택 UI (바텀시트)
        - 바텀시트 상단에 **검색바** 제공 (placeholder : "은행명을 검색해주세요") — 은행명 입력 시 목록이 실시간 필터링되어 검색으로도 선택 가능, 검색 결과 없음 시 빈 상태 문구 노출
        - 계좌번호 입력란 : 숫자·하이픈 입력 (placeholder : "계좌번호를 입력해주세요")
        - 저장 규칙 : DB에는 은행명이 아닌 **금융결제원 표준 은행 코드**로 저장하고, 화면 노출 시 코드 → 은행명/로고로 맵핑
        - 지원 은행/코드 : KB국민(004), 신한(088), 우리(020), 하나(081), NH농협(011), IBK기업(003), 카카오뱅크(090), 토스뱅크(092), 케이뱅크(089), SC제일(023), 우체국(071), 새마을금고(045), 신협(048), 부산(032), iM뱅크(031), 경남(039), 광주(034), 전북(037), 제주(035), 수협(007), KDB산업(002)
- 총액 입력 필드 (placeholder : "총 결제 금액을 입력해주세요") + 분배 방식 토글 [N분의 1로 나누기] / [금액 직접 입력]
    - 총액 입력 필드 **왼쪽**에 [계산기 사용] 버튼을 **상시 노출** — 사용자가 별도 계산기 앱을 켜기 전에 이 서비스 안에서 계산 기능을 쓸 수 있음을 인지하도록 함. 버튼 클릭 시 계산기가 열리고, 계산 결과는 총액 필드에 자동 입력
    - 분배 방식 보조 문구
        - [N분의 1로 나누기] : 참여자 수에 맞춰 자동으로 나눠드려요
        - [금액 직접 입력] : 사람마다 다른 금액을 직접 입력할 수 있어요
    - [금액 직접 입력] 토글 클릭 시 인터랙션 : "함께 정산할 사람" 첫 번째 이름 입력창에 자동 포커스(커서 깜빡임)되고, 해당 섹션이 화면 중앙에 오도록 부드럽게 스크롤(smooth scroll, center 정렬)
- 참여자 이름 입력 (동적 추가/삭제 가능한 리스트)
    - 요소별 문구
        - 섹션 제목 : 함께 정산할 사람
        - 입력 placeholder : 이름을 입력해주세요
        - 추가 버튼 : + 참여자 추가
        - 삭제 버튼 : 삭제
        - 빈 상태 문구 : 정산할 사람을 1명 이상 추가해주세요
    - CTA 비활성화 및 안내 문구
        - 총 금액 미입력 -> 정산할 금액을 입력하면 요청을 만들 수 있어요
        - 참여자 0명 -> 함께 정산할 사람을 1명 이상 추가해주세요
- 절삭 토글 카드 (기본 on — 배려 모드를 기본 경험으로 제공, 원치 않는 사용자는 끌 수 있고 변경 값은 전역 설정으로 유지)
    - 요소별 문구
        - 카드 제목 : 빠른 정산 배려 모드
        - 카드 본문 : 빠르게 확인해준 사람에게 끝자리를 조금 덜어 요청할 수 있어요
        - 아래 회식 미리보기 : 예) 26,437원 → 26,000원만 보내주세요 :)
        - 보조 설명 : 요청마다 고민하지 않아도, 정해진 기준으로 자동 적용돼요
- (재사용자에게만) 상단 배너
    - 요소별 문구
        - 배너 제목 : 지금까지 받은 배려
        - 배너 문구 : 친구들이 지난 정산에서 총 1,450원을 덜어줬어요
        - CTA 문구 : 나도 배려 모드 써보기
    - 배너 노출 조건
        - 노출
            - 과거 절삭 정산 받은 기록 있음
            - 빠른 정산 배려 모드 OFF 상태
        - 비노출
            - 빠른 정산 배려 모드 ON 상태
            - 절삭 기록 없음

**CTA / 인터랙션:**

- [정산 요청하기] — 클릭 시 참여자별 고유 링크가 생성되며 3-2 화면으로 이동. 버튼은 참여자 1명 이상+총액 입력 전까지 비활성화

### 3-2. 정산 요청 공유

**화면 목적:** 정산 요청자가 참여자별 개별 링크를 확인하고, 각 참여자에게 맞는 링크를 공유하는 화면

**핵심 문구:** 타이틀 "정산 요청이 준비됐어요" / 서브 "각자 이름에 맞는 링크를 보내주세요"

**주요 요소:**

- 참여자별 카드 리스트 (이름 + 금액 + 개별 링크)
    - 구성 요소
        - 참여자 이름
        - 정산 금액 (기본 금액 또는 예상 금액)
        - 참여자별 고유 링크
- 각 카드 하단 [카톡으로 보내기] [링크 복사] 버튼
    - 예시

        > 민지님에게 보낼 정산 요청
        >
        > 기본 정산 금액
        > 26,437원
        >
        > 빠르게 확인하면
        > 26,000원까지 줄어들 수 있어요
        >
        > [카카오톡으로 보내기] [링크 복사]

- 공유는 **참여자별 개별 발송만 지원** — 전체 링크 일괄 복사 기능은 제공하지 않음
    - 이유 : 여러 링크를 한 메시지로 보내면 다른 사람이 남의 링크를 잘못 클릭할 수 있고, 이 경우 당사자가 보지 않았는데 열람(확인함) 처리되어 절삭 판정까지 확정되는 오류가 발생함

**CTA / 인터랙션:**

- 카카오톡으로 보내기 : KakaoTalk 공유 API 호출
    - 공유 실패 시 : 해당 참여자 링크와 문구를 클립보드에 복사하고 토스트 노출
- 링크 복사 : 참여자별 링크 복사
- 나중에 보낼게요 : 3-4 정산 현황 대시보드로 이동

### 3-3. 정산 확인 (참여자 View) — 지불자가 링크 접속 시

#### (신규 브라우저인 경우만) 진입 전 단계

**화면 목적:** 참여자가 과거 기록과 연결될 수 있도록 선택적으로 계좌 뒤 4자리를 입력하는 화면

**핵심 문구:** "처음 접속한 기기 같아요!" / 서브 "다음에도 배려 기록을 이어볼 수 있게, 자주 쓰는 계좌 뒤 4자리를 남길 수 있어요 (선택)"

**주요 요소:**

- 입력 필드별 문구
    - 라벨 : 계좌번호 뒤 4자리
    - Placeholder : 숫자 4자리 입력
    - 보조 문구 : 입력하지 않아도 정산 금액은 확인할 수 있어요
    - Skip 링크 : 건너뛰기

**CTA:** [계속하기] 또는 [건너뛰기]

- 계속하기 : 4자리 입력값 저장 후 결과 화면으로 이동
- 건너뛰기 : 식별자 연결 없이 결과 화면으로 이동
    - 보조 문구: 건너뛰어도 정산 확인은 가능하지만, 이전 배려 기록과는 연결되지 않아요.
- 4자리 미입력 후 계속하기 : 버튼 비활성화

#### 결과 화면

**화면 목적:** 참여자가 본인의 정산 금액, 절삭 적용 여부, 계좌 정보를 확인하고 외부 앱에서 송금할 수 있게 돕는 화면

**핵심 문구:**

- 절삭 적용 시: "빠르게 확인해주셨네요 :)" + "끝자리는 요청자가 덜어줬어요!" + 금액 카드 "26,000원만 보내면 돼요\n 기존 금액 26,437원에서 437원이 덜어졌어요."
- 절삭 미적용 시: "정산 금액을 확인해주세요" + 금액 카드 "26,437원\n 아래 계좌로 직접 송금해주세요!"

**필수 요소:**

- 요청자 이름
- 참여자 이름
- 확정 금액 카드
- 원래 금액
- 절삭 금액
- 입금 계좌 정보 — 은행 로고·은행명 + 계좌번호 (DB의 은행 코드를 은행명/로고로 맵핑해 노출, 계좌번호는 요청자가 입력한 값)
- 계좌 복사 버튼
- 안내 문구 노출

**CTA / 인터랙션:**

- [계좌번호 복사하기] — 클릭 시 클립보드 복사 + "계좌번호를 복사했어요." 안내 토스트.

### 3-4. 정산 현황 (Dashboard) — 요청자 재방문 시

**화면 목적:** 정산 요청자가 참여자별 링크 확인 여부와 확정 금액을 확인하고, 미확인 참여자에게 같은 링크를 다시 공유할 수 있는 화면

**핵심 문구:** 타이틀 "정산 현황" / 서브 "누가 금액을 확인했는지 한눈에 볼 수 있어요"

**주요 요소:**

- 참여자별 카드 구성요소
    - 참여자 이름
    - 상태 뱃지 (미확인/확인함)
    - 확정 금액
    - 절삭 여부
    - 확인 시간
    - CTA : 미확인 상태일 때만 [다시 공유] -> 같은 링크 재공유
- 하단 섹션 "배려 기록"
    - 섹션 설명 : 이번 정산까지 내가 덜어준 금액을 볼 수 있어요
    - 참여자별 문구 : 지금까지 {이름}님에게 {총액}원을 덜어줬어요
    - 안내 문구 : 이 기록은 나에게만 보여요.
    - 노출 조건
        - 노출
            - 요청자 본인 대시보드
            - 절삭 기록이 1건 이상 있음, 없으면 빈 상태 문구 노출 (아직 배려 기록이 없어요.\n빠르게 확인한 친구에게 덜어준 금액이 생기면 여기에서 볼 수 있어요.)

**CTA / 인터랙션:**

- 새 정산 만들기 : 3-1 정산 요청 생성 화면으로 이동
- 기존 사용자예요! : 글로벌 토글 설정과 받은 배려 배너 반영
