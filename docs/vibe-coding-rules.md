## 0. 기술 스택 및 구현 원칙 (AI 필독)

> 💡 이 섹션은 AI의 구현 오류를 막는 안전장치입니다. 프로젝트마다 값만 바꾸고 항목은 유지하세요.

- **프레임워크**: {{예: Next.js 15 (App Router) + React 19 + TypeScript}}
- **스타일링**: {{예: Tailwind CSS v4}}
- **백엔드/DB**: {{예: Supabase / 없음(정적 사이트) / 로컬 스토리지만 사용}}
- **배포 대상**: {{예: Vercel / GitHub Pages / 로컬 실행만}}
- **구현 원칙 (그대로 유지 권장)**:
    - 외부 API·DB 호출은 실패해도 앱이 죽지 않도록 폴백(fallback) 처리할 것.
    - 동적 라우트, 상태 관리 등은 위 프레임워크의 **최신 문법 기준**으로 작성할 것.
    - 비밀 키(Secret Key)는 절대 클라이언트 코드나 채팅에 노출하지 말 것. (공개용 Publishable/Anon Key는 예외)
    - 모든 화면은 명세에 없는 상태(로딩 / 빈 데이터 / 에러)도 자연스럽게 처리할 것.
