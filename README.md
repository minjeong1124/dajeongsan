# 다정산 (Dajeongsan)

> 금액은 정확하게, 요청은 조금 더 다정하게.
> 정산 요청을 "독촉"이 아니라 "배려"로 바꾸는 정산 서비스 — 메이커톤 테스트 사이트

## 실행 방법

```bash
npm install
npm run dev
# http://localhost:3000
```

## 기술 스택

- Next.js 15 (App Router) + React 19 + TypeScript
- Tailwind CSS v4
- Supabase (실패 시 localStorage 폴백 — 앱이 죽지 않는 원칙)

## Supabase 설정

연동 정보(docs/supabase-info.md의 Project ID·Publishable Key)는 `lib/supabase.ts`에
직접 입력되어 있어 별도의 환경 변수 파일이 필요 없습니다.

- **최초 1회**: [database/schema.sql](database/schema.sql) 전체를 Supabase 대시보드 >
  SQL Editor에 붙여넣고 실행하면 테이블 생성 + 실시간(Realtime) 구독까지 활성화됩니다.
- 요청 생성·링크 열람 시 데이터가 즉시 적재되고, 요청자 대시보드는 참여자의 열람을
  Realtime 구독으로 실시간 렌더링합니다.
- 테이블이 없거나 네트워크가 끊겨도 localStorage 폴백으로 전체 플로우가 동작합니다.

## 화면 구성 (docs/screen-specifications.md)

| 라우트 | 화면 |
| --- | --- |
| `/` | 3-1. 정산 요청 생성 — 총액/N분의1·직접입력, 참여자 리스트, 절삭 토글 카드, 재사용자 배려 배너, 인라인 계산기 |
| `/share/[id]` | 3-2. 정산 요청 공유 — 참여자별 링크 카드, 카톡 공유(실패 시 클립보드 폴백), 문구 일괄 복사 |
| `/p/[pid]` | 3-3. 정산 확인(참여자) — 신규 브라우저 계좌 뒷자리 게이트, 절삭 판정 결과, 계좌 복사 |
| `/dashboard/[id]` | 3-4. 정산 현황 — 참여자별 열람 상태/확정 금액, 다시 공유, 배려 기록 |

## 핵심 로직 (docs/project-outline.md · project-hypothesis.md)

- **시간 기반 절삭**: 링크 최초 열람 시각 기준 — 1시간 이내 1,000원 미만 절삭(최대 999원),
  5시간 이내 100원 미만 절삭(최대 99원), 이후 절삭 없음. 판정 결과는 최초 열람 시 확정·저장됩니다.
- **소액 보호**: 5,000원 미만 정산은 절삭을 생략합니다. (가설 문서의 리스크 대응 반영)
- **절삭 토글 전역 설정**: 기본 off, 한 번 켜면 브라우저에 유지됩니다.
- **사용자 식별**: 브라우저 익명 토큰 + (신규 브라우저) 계좌 뒷자리 4자리 매칭.
- **배려 기록**: 요청자 대시보드에 관계(이름)별 누적 절삭 금액, 재방문 생성 화면에
  "지금까지 받은 배려" 배너로 상호성을 노출합니다.

## 절삭 시나리오 테스트 팁

절삭 등급은 요청 생성 시각 대비 경과 시간으로 판정되므로, 로컬 테스트에서는
요청 생성 직후 참여자 링크를 열면 1시간 이내 등급(999원 미만 절삭)이 적용됩니다.
